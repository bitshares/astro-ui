**Barter** lets two parties agree on a direct asset-for-asset trade without a public order book. One side drafts the trade as a `proposal_create` whose `proposed_ops` contain the `transfer` operations moving assets between the parties. The counterparty reviews and approves the proposal. An optional **escrow** mode routes the assets through a neutral escrow account.

**Open in the app:** [Barter](/barter.html).

## What it does

<CardGrid>
  <Card title="Private trades" icon="lock">
    No public order book — only the two (or three, with escrow) parties see the terms.
  </Card>
  <Card title="Proposal-based" icon="document">
    Trades are `proposal_create` ops, so both sides must approve before anything moves.
  </Card>
  <Card title="Escrow option" icon="shield">
    Route funds through an escrow account for trust-minimized settlement.
  </Card>
</CardGrid>

## How to use

<Steps>

1. Open **Barter** from the homepage under *Exchanging Funds*.

2. Add the lines you will give and the lines you want to receive (asset + amount + counterparty account).

3. Optionally enable **escrow** and pick an escrow account; a small BTS escrow fee is added.

4. Submit. The app wraps your transfers in a `proposal_create` with a 7-day expiration and a 1-hour review period.

5. The counterparty opens the proposal, reviews the `proposed_ops`, and approves. On approval the `transfer` ops execute.

</Steps>

<Aside type="note" title="Proposal timing">
  The proposal expires 7 days after creation (`expiration_time` = now + 7 days) and carries a `review_period_seconds` of `3600`, giving the reviewer a 1-hour inspection window before approval takes effect.
</Aside>

## Operation used

Barter emits a `proposal_create` (operation `22`) whose `proposed_ops` contain `transfer` operations (operation `0`):

```json
{
  "fee": { "amount": 0, "asset_id": "1.3.0" },
  "fee_paying_account": "1.2.x",
  "expiration_time": "2026-07-25T12:00:00",
  "proposed_ops": [
    {
      "op": [
        0,
        {
          "fee": { "amount": 0, "asset_id": "1.3.0" },
          "from": "1.2.x",
          "to": "1.2.x",
          "amount": { "amount": 10000000, "asset_id": "1.3.0" },
          "extensions": {}
        }
      ]
    },
    {
      "op": [
        0,
        {
          "fee": { "amount": 0, "asset_id": "1.3.0" },
          "from": "1.2.x",
          "to": "1.2.x",
          "amount": { "amount": 2500000, "asset_id": "1.3.x" },
          "extensions": {}
        }
      ]
    }
  ],
  "review_period_seconds": 3600,
  "extensions": {}
}
```

| Field | Meaning |
| --- | --- |
| `fee_paying_account` | Account that pays the proposal fee (peer, or the escrow account in escrow mode). |
| `expiration_time` | ISO-8601; proposal becomes invalid after this. |
| `proposed_ops` | Array of `{ op: [opId, opBody] }` — here the `transfer` ops. |
| `review_period_seconds` | Inspection window before approval is actionable. |

Each nested `transfer` carries `from`, `to`, and `amount` (`{ amount, asset_id }`).
