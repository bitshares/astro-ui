**Proposals** lists the `proposal` objects relevant to an account — including the `proposed_ops` they contain (which may be `transfer`, `account_update`, and more). You can inspect each operation, then **approve** (adding your authority to the proposal) or **reject** (delete it).

**Open in the app:** [Proposals](/proposals.html).

## What it does

<CardGrid>
  <Card title="Inspect" icon="document">
    Read each proposal's wrapped operations.
  </Card>
  <Card title="Approve" icon="check">
    Add your approval via `proposal_update`.
  </Card>
  <Card title="Reject" icon="close">
    Delete a proposal via `proposal_delete`.
  </Card>
</CardGrid>

## How to use

<Steps>

1. Open **Proposals** from the homepage under *Account Overviews*.

2. Select the account. Each `1.20.x` proposal shows its proposed operations and required approvals.

3. Individually review/approve the contained operations, then **Approve** the whole proposal → `proposal_update`.

4. To discard a proposal, **Reject** it → `proposal_delete`.

</Steps>

<Aside type="note" title="Approval model">
  Approval is additive — `proposal_update` adds your account/key to the approval sets. There is no separate `proposal_vote`; the proposal executes once all required approvals are gathered (after any review period).
</Aside>

## Operations used

<Tabs>
  <TabItem label="proposal_create">

Creating a proposal wraps one or more operations in a `proposal_create` (operation `22`). The proposal executes once all required approvals are gathered and the review period has elapsed. Used by [Timed Transfer](/docs/transfer/timed_transfer.html) and [Barter](/docs/exchanging/barter.html).

```json
{
  "fee_paying_account": "1.2.x",
  "expiration_time": "2026-07-25T12:00:00.000Z",
  "proposed_ops": [
    {
      "op": [
        0,
        {
          "fee": { "amount": 0, "asset_id": "1.3.0" },
          "from": "1.2.x",
          "to": "1.2.x",
          "amount": { "amount": 50000, "asset_id": "1.3.0" },
          "extensions": {}
        }
      ]
    }
  ],
  "review_period_seconds": 3600,
  "extensions": {}
}
```

| Field | Meaning |
| --- | --- |
| `fee_paying_account` | Account that pays the proposal fee. |
| `expiration_time` | ISO-8601; proposal becomes invalid after this. |
| `proposed_ops` | Array of `{ op: [opId, opBody] }` — the operations to execute. |
| `review_period_seconds` | Inspection window before the operations are actionable. |

  </TabItem>
  <TabItem label="proposal_update (approve)">

```json
{
  "fee_paying_account": "1.2.x",
  "proposal": "1.20.x",
  "active_approvals_to_add": ["1.2.x"],
  "owner_approvals_to_add": [],
  "key_approvals_to_add": [],
  "active_approvals_to_remove": [],
  "owner_approvals_to_remove": [],
  "key_approvals_to_remove": [],
  "extensions": {}
}
```

  </TabItem>
  <TabItem label="proposal_delete (reject)">

```json
{
  "fee_paying_account": "1.2.x",
  "proposal": "1.20.x",
  "using_owner_authority": false,
  "extensions": {}
}
```

  </TabItem>
</Tabs>

| Field | Meaning |
| --- | --- |
| `proposal` | The `1.20.x` proposal object id. |
| `fee_paying_account` | Your account paying the fee. |
| `*_approvals_to_add` | Your account/key ids being added to the approval set. |
| `using_owner_authority` | Whether owner (vs active) authority signs the deletion. |
