**Airdrop Calculator** distributes an asset to many accounts in a single workflow. You build a recipient list — manually, from ticket locks, call-order collateral, top holders, limit-order activity, or lifetime-member upgrades — run a deterministic selection algorithm, then broadcast the resulting transfers in batches. Under the hood it's simply many `transfer` operations grouped per transaction.

**Open in the app:** [Airdrop Calculator](/airdrop_calculate.html).

## What it does

<CardGrid>
  <Card title="Bulk distribution" icon="send">
    Send the same or custom amounts to hundreds of accounts.
  </Card>
  <Card title="Deterministic selection" icon="cpu">
    Choose winners via seeded algorithms (freebie, forward, pi, alien blood, …).
  </Card>
  <Card title="Batched broadcast" icon="rocket">
    Transfers are chunked into transactions to fit chain limits.
  </Card>
</CardGrid>

## How to use

<Steps>

1. Open **Airdrop Calculator** from the homepage under *Transferring Funds*.

2. Build a candidate pool — manual input, ticket locks, call-order collateral, top holders, limit-order activity, or LTM upgrades.

3. Pick a distribution mode (equal split, fixed amount, or custom) and a selection algorithm seeded from a block witness signature.

4. Review the winning accounts and per-account amounts.

5. Broadcast. The app slices recipients into batches, each a transaction of `transfer` ops, signed via your Beet wallet.

</Steps>

<Aside type="note" title="Batch sizing">
  Each transaction carries one batch of transfers; the app refuses to build a batch larger than the chain's `maximum_transaction_size`. Very large airdrops therefore span many transactions.
</Aside>

## Operation used

Every recipient becomes a `transfer` (operation `0`). A single batch transaction looks like:

```json
[
  {
    "fee": { "amount": 0, "asset_id": "1.3.0" },
    "from": "1.2.x",
    "to": "1.2.x",
    "amount": { "amount": "1000", "asset_id": "1.3.0" },
    "extensions": {}
  },
  {
    "fee": { "amount": 0, "asset_id": "1.3.0" },
    "from": "1.2.x",
    "to": "1.2.x",
    "amount": { "amount": "2500", "asset_id": "1.3.0" },
    "extensions": {}
  }
]
```

| Field | Meaning |
| --- | --- |
| `from` | Your account (`1.2.x`). |
| `to` | Each recipient account (`1.2.x`). |
| `amount` | Integer amount (× precision) for that recipient. |
