**Timed Transfer** schedules a transfer to occur at a future time. Instead of broadcasting a `transfer` directly, the app wraps it in a `proposal_create`. The embedded `transfer` only executes once the proposal's review period has elapsed and the proposal is approved — giving you a delayed, reviewable send.

**Open in the app:** [Timed Transfer](/timed_transfer.html).

## What it does

<CardGrid>
  <Card title="Delayed send" icon="clock">

## How to use

<Steps>

1. Open **Timed Transfer** from the homepage under *Transferring Funds*.

2. Choose the asset, amount and recipient.

3. Set the execution time — the app sets `expiration_time` to that moment and a `review_period_seconds` window before it.

4. Authorize in your Beet wallet. The app broadcasts a `proposal_create` wrapping a `transfer`.

</Steps>

<Aside type="note" title="How the delay works">
  The inner `transfer` (operation index `0`) does not execute on broadcast. It executes when the proposal is approved and its `review_period_seconds` has passed, no later than `expiration_time`. Set `expiration_time` comfortably after the review period.
</Aside>

## Operation used

Timed Transfer emits a `proposal_create` (operation `22`) whose single `proposed_ops` entry is a `transfer` (operation `0`):

```json
{
  "fee_paying_account": "1.2.x",
  "expiration_time": "2026-07-25T12:00:00.000Z",
  "proposed_ops": [
    {
      "op": [
        0,
        {
          "fee": { "amount": 0, "asset_id": "1.3.0" },
          "from": "1.2.x",
          "to": "1.2.x",
          "amount": { "amount": 50000, "asset_id": "1.3.0" },
          "extensions": {}
        }
      ]
    }
  ],
  "review_period_seconds": 3600,
  "extensions": {}
}
```

| Field | Meaning |
| --- | --- |
| `fee_paying_account` | Account that pays the proposal fee (usually the recipient). |
| `expiration_time` | ISO-8601; proposal becomes invalid after this. |
| `proposed_ops` | Array of `{ op: [opId, opBody] }` — here the wrapped `transfer`. |
| `review_period_seconds` | Inspection window before the transfer is actionable. |
